# Document Analysis

## When to use
Use this skill when the user wants to analyze, summarize, or extract information from documents.

## Instructions
1. Read the entire document before starting analysis
2. Identify the document type (report, specification, article, etc.)
3. Extract key sections: title, authors, dates, main findings
4. Create a structured summary with:
   - One-paragraph executive summary
   - Key findings or main points (bulleted)
   - Notable data or statistics
   - Conclusions or recommendations
5. For technical documents, note any diagrams, tables, or code referenced
6. Flag sections that are unclear or potentially outdated
